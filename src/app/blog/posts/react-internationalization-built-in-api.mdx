## Introduction

Managing multilingual support (internationalization, i18n) is crucial for global applications. React now simplifies this process with new built-in APIs designed specifically for internationalization, reducing complexity and enhancing maintainability.

Let's dive into how to effectively leverage these new features.

## Using React’s Built-in Internationalization APIs

React’s built-in internationalization APIs offer streamlined, performant solutions without relying on external libraries.

Here's how you use the new built-in API for basic translations:

<CodeBlock
    marginBottom="16"
    codeInstances={[
      {
        code:
`import { useTranslation } from 'react/i18n';

function Greeting() {
  const { t } = useTranslation();

  return <h1>{t('welcomeMessage')}</h1>;
}

export default Greeting;`,
        language: "jsx",
        label: "src/components/Greeting.jsx"
      }
    ]}
/>

## Implementing Language Switching

Effortlessly switch languages using React's built-in context and hooks:

<CodeBlock
    marginBottom="16"
    codeInstances={[
      {
        code:
`import { useLanguage } from 'react/i18n';

function LanguageSwitcher() {
  const { language, setLanguage } = useLanguage();

  return (
    <select value={language} onChange={(e) => setLanguage(e.target.value)}>
      <option value="en">English</option>
      <option value="fr">French</option>
      <option value="es">Spanish</option>
    </select>
  );
}

export default LanguageSwitcher;`,
        language: "jsx",
        label: "src/components/LanguageSwitcher.jsx"
      }
    ]}
/>

## Advantages of Built-in Internationalization APIs

### 1. Improved Performance
Built-in solutions are optimized directly within React, reducing overhead and boosting efficiency.

### 2. Simplified Integration
No extra libraries or external configuration required, reducing complexity and dependency issues.

## Handling Dynamic Content

React's built-in APIs smoothly handle dynamic translation scenarios such as interpolation and pluralization:

<CodeBlock
    marginBottom="16"
    codeInstances={[
      {
        code:
`import { useTranslation } from 'react/i18n';

function ItemCount({ count }) {

  return <p>{t('items', { count })}</p>;
}

export default ItemCount;`,
        language: "jsx",

      }

## Practical Example: Setting Up Internationalization Provider

Here's how to configure your app with React's built-in i18n provider for complete language management:

<CodeBlock
    marginBottom="16"
    codeInstances={[
      {
        code:
`import { I18nProvider } from 'react/i18n';
import App from './App';

const translations = {
  en: { welcomeMessage: "Welcome!", items: "{{count}} items" },
  fr: { welcomeMessage: "Bienvenue!", items: "{{count}} articles" },
};

function Root() {
  return (
    <I18nProvider translations={translations} defaultLanguage="en">
      <App />
    </I18nProvider>
  );
}

export default Root;`,
        language: "jsx",
        label: "src/Root.jsx"
      }
    ]}
/>

## Conclusion

React’s built-in internationalization APIs simplify managing multilingual applications, reducing external dependencies and improving application maintainability and performance. Incorporate these APIs into your React projects to streamline internationalization effortlessly.
